Themes

https://bookdown.org/yihui/blogdown/other-themes.html https://themes.gohugo.io/

Hugo generates sites using ‘themes’ aka rendering templates. There are many themes available on Hugo, however some of them must be modified to be rendered by rmarkdown. There is plenty of documentation if you find a theme that must be modified to work.

After choosing a theme you don’t need to install it by hand, it is better if you pass the theme name to blogdown::new_site() or by building a new site in an RStudio project context (shown below).

New Project

https://support.rstudio.com/hc/en-us/articles/200526207-Using-Projects https://bookdown.org/yihui/blogdown/rstudio-ide.html

You don’t have to start a new project in R Studio to build a site, but it keeps everything much more organized if you do.

File -> New Project -> New Directory then select “Website using blogdown” This is where you can name you new directory and set your Hugo theme.

The blogdown author recommends enabling the option “Preview site after building” and disabling the option “Re-knit current preview when supporting files change”. To find those options look on R Studio under Tools -> Project Options...-> Build Tools

Publishing for the first time

Restart R

blogdown::hugo_build() # This will build the site as if from scratch, all webpages will be rendered

After making new pages or any changes

blogdown::serve_site() # This will only render the webpages that have been changed

Get netlify and GitHub setup

https://github.com/ https://www.netlify.com/ # can use GitHub account to sign up for Netlify

On GitHub create a new repository for you site. Make sure that it is Public.

Now that your website is hosted on GitHub, go on Netlify New Site from Git -> Continuous Deployment [GitHub] -> Pick a Repository -> ... The default domain name is <name.netlify.app> but there are options to purchase or set up your a domain name.

Update config.toml with domain name

Use the domain name generated by Netlify, or use the custom domain name that you setup in Netlify.

In config.toml update the parameter at the top baseurl = "https://my-website.netlify.app/"

Don’t forget to commit and push your updated website to GitHub.

git add .
git status
git commit -m "updated baseurl"
git push -u origin main

Now wait a few seconds, check your Netlify website, fix bugs, edit CSS, make new posts
